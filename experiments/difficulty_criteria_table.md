# 实验样本难度判定标准表（difficulty）

> 适用范围：`experiments/eval_dataset_v*.jsonl` 中的 `difficulty` 字段标注。  
> 目标：保证不同实验组在同一难度分层下可公平对比。

## 1. 难度等级定义

| 等级 | 定义 | 典型特征 | 建议标签值 |
|---|---|---|---|
| 低难（Easy） | 事故语义清晰，风险类型单一，处置主线明确 | 信息完整、约束冲突少、检索证据易命中 | `easy` |
| 中难（Medium） | 存在信息缺口或并发风险，需要补全推理 | 2类风险并发、主体不全、时序不完全明确 | `medium` |
| 高难（Hard） | 描述模糊/冲突、多风险耦合，需复杂约束协同 | 语义歧义、多部门联动、强时空约束、多模态依赖 | `hard` |

## 2. 判定维度与打勾规则

每条样本按下列维度勾选“是否命中”，命中越多难度越高。

| 维度 | 命中条件（任一满足即勾选） |
|---|---|
| 信息完整性不足 | 关键字段缺失（如伤亡/泄漏/起火状态不明） |
| 风险并发性 | 同时出现 2 类及以上风险（如“被困+泄漏”“火情+交通中断”） |
| 约束复杂度 | 存在明显先后依赖或安全门槛（先封控再处置、先评估再通行） |
| 主体协同复杂度 | 至少需要 2 个以上部门联动且角色边界不清晰 |
| 证据检索难度 | 需要跨图谱约束与文本证据联合才能支持决策 |
| 表达噪声 | 口语化、错别字、矛盾描述、信息冗余明显 |
| 多模态依赖 | 仅文本不足，需结合图片/现场特征判断 |

## 3. 推荐判定阈值（可直接执行）

- 命中 0~1 项：标记 `easy`
- 命中 2~3 项：标记 `medium`
- 命中 ≥4 项：标记 `hard`

> 若出现“关键安全信息缺失 + 多风险并发”组合，优先上调一级难度。

## 4. 交通应急场景示例

### Easy 示例
- 文本："雨天高速追尾，1人轻伤，无明火，车道受阻。"
- 判定：信息完整、单一主风险、处置链路明确 → `easy`

### Medium 示例
- 文本："隧道内多车追尾，疑似泄漏，部分人员被困。"
- 判定：风险并发（泄漏+被困），且信息不完整 → `medium`

### Hard 示例
- 文本："夜间桥梁路段客货车相撞，现场冒烟，通信描述前后不一致。"
- 判定：表达噪声+高风险场景+多部门协同+潜在多模态依赖 → `hard`

## 5. 标注一致性建议（论文可写）

1. 两名标注者独立标注 difficulty。  
2. 分歧样本通过规则复审，不直接主观拍板。  
3. 记录一致性指标（如 Cohen's Kappa），建议 ≥0.75。  
4. 若暂无法双人标注，至少保留“规则命中项”字段，便于审稿复核。
