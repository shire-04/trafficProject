# G5提分改动计划（执行版）

更新时间：2026-04-04
目标：在不作弊前提下，让 G5 在同口径 50 样本下稳定不低于 G4，并逐步反超。

## 1. 约束与成功标准

- 固定运行环境：`traffic_env`
- 固定评测口径：`EVAL_SCORE_BACKEND=llm`，评委模型与权重保持一致
- 固定对比方式：同样本 `baseline vs candidate`
- 成功标准：
  - 10 样本快筛：G4 不降，G5 至少一项核心维度上升
  - 50 样本门禁：G5 总分不低于 G4，且稳定性指标不下降

## 2. 改动分期

### Phase A（已开始）最小侵入提分

1. 修订目标结构化（已完成）
- 文件：`src/agents.py`
- 改动：在 `CommanderAgent._revise_with_llm` 增加 `structured_weaknesses`，把 `review_reason/missing_action/risk_note` 映射为结构化修订目标，加入 `mandatory_fix_actions` 与图谱动作提示。
- 目的：减少“泛改写”，提升长尾样本的定向修复能力。

2. 弱步骤增强去误伤（已完成）
- 文件：`src/agents.py`
- 改动：优化 `_strengthen_steps_with_context_actions`，仅重写“明确弱步骤”，对已有责任主体+动作的步骤不再替换。
- 目的：减少重复模板句，保护 G4，同时让 G5 修订更稳。

### Phase B（下一步）修订守卫细化

1. 质量守卫阈值参数化（待实施）
- 文件：`src/orchestrator.py`
- 改动：将固定阈值改为可配置，并按模式设默认值（G5 更关注防回归，G4 更关注单轮可执行）。

2. 修订改写比例限制（待实施）
- 文件：`src/orchestrator.py`
- 改动：新增“改写比例过高且质量未提升”回退规则。

### Phase C（待实施）评审反馈质量提升

1. 失败类型映射标准化（待实施）
- 文件：`src/agents.py`
- 改动：对 `failure_type` 和审查理由做更稳定的标签归一，减少噪声反馈。

2. 规则审查轻量增强（待实施）
- 文件：`src/agents.py`
- 改动：仅补充高风险漏项识别，不改变评测口径。

## 3. 验证矩阵

1. 冒烟验证
- `Tools/check_pipeline_regression.py`
- 要求：5/5 成功，且无异常退出

2. 快速门禁（10 样本）
- 组别：G4 与 G5
- 目标：筛掉会伤 G4 或对 G5 无收益的改动

3. 正式门禁（50 样本）
- 组别：G5（必要时补 G4 对照）
- 输出：`total/safe/comp/exec` + 稳定性指标（judge 成功率、异常率）

## 4. 回滚策略

- 每轮只合入一个主杠杆（生成 / 修订 / 守卫）
- 若 10 样本不达标，直接回滚本轮改动
- 若 50 样本不达标，回滚并保留诊断数据用于下轮
